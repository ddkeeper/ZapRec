# 画中画 (PIP) 模块问题诊断与交互重构规范 (完整版)

## 一、 问题诊断 (Bug & Issue Diagnosis)

当前画中画及多窗口协同模块存在以下底层逻辑与事件处理缺陷，需进行技术修复：

1. **事件拦截与热区冲突 (Event & Z-index Conflicts)**
   * 中心“形状切换”按钮的触发层级覆盖了边缘的缩放热区。表现为：光标已变为缩放双箭头，但点击/拖拽操作被中心按钮拦截，导致缩放失效。
   * 鼠标悬停在 PIP 内部主体区域时，未正确绑定或透传拖拽（Drag）事件，导致小窗无法移动。
2. **视觉遮罩与逻辑边界溢出 (Masking & Hitbox Overflow)**
   * 圆形形态未严格执行边界裁剪，当打开新窗口时，画中画小窗置于新窗口上会暴露出淡色的矩形四角残影。
   * 交互逻辑未跟随形态切换：圆形状态下，圆弧外围会有几个矩形角，仍能响应缩放事件，产生严重的交互割裂感。
3. **工具条只读状态语义模糊 (Toolbar State Semantics)**
   * 录制过程中，工具条缺乏清晰的“只读且锁定”表现机制，未能正确维持录制前的“开启（高亮）/关闭（置灰）”视觉标识，极易引起状态歧义。
4. **圆形/矩形缩放的控制有问题**
* 控制点在新打开的背景为白色窗口下几乎看不清。
* **矩形形态（8 视口控制点）**
  * **四角定点（左上/左下/右上/右下）**：严格锁定当前画面的纵横比进行整体等比缩放，确保画面主体不变形。
  * **四边中点（上/下/左/右）**：执行横向或纵向的单向拉伸。该操作实为**改变视口裁剪区 (Viewport Cropping)**，需配合自适应填充（`object-fit: cover`），允许画面自由裁切为竖屏或宽屏，而非像素拉伸畸变。
* **圆形形态（4 对角控制点）**
  * **锚点重构**：彻底废除矩形边角，控制点精确吸附于圆弧的四个对角（45°、135°、225°、315°），控制点。
  * **强制等比**：拖拽任意对角控制点，必须保持圆心相对稳定，强制以 1:1 绝对等比例放大或缩小，杜绝任何椭圆形变。

4. **透明遮罩层事件穿透与光标闪烁异常 (Cursor Flickering)**
    * 在“区域录制”模式下，鼠标在主工具条信息源开关区及 PIP 边缘移动时，光标在“禁止图标”与“正常/缩放指针”之间高频闪烁（全屏模式下无此现象）。此为典型的多窗口叠加状态下，透明区域遮罩层与底层 UI 的事件捕获/冒泡冲突（Z-index 或 Pointer-events 打架），导致系统频繁重绘鼠标状态。

## 二、 交互体验规范 (UX & Interaction Blueprint)

为确保交互的确定性、丝滑感与系统稳健性，重构后的模块需严格遵循以下产品级标准：

### 1. 纯净视觉与区域级触发 (Visual Purity & Hover Trigger)
* **静默状态**：鼠标未移入 PIP 区域时，画面必须为边缘锐利的绝对纯净图形（无控制点、无虚线框、无背景残影），实现 100% 沉浸。
* **按需显现**：仅当鼠标**真正移入** PIP 内部，对应的控制点与中心形状按钮才平滑淡入；移出则即刻消散。

### 2. 交互图层严格隔离与光标防抖 (Hitbox Isolation & Cursor Stability)
* **区域录制防抖**：在区域录制模式下，必须严格界定选区遮罩层与 UI 控件层的层级关系，剥夺非必要图层的鼠标事件响应权限（`pointer-events: none`），确保光标状态平滑稳定，彻底消除闪烁。
* PIP 窗口交互区在逻辑上划分为三个绝对互不干扰的层级：
  * **中心触发层**：放置形状切换按钮，需限定极小且精确的热区，严禁拦截外围事件。
  * **主体移动层**：除按钮与边缘热区外的所有内部画面区域，按下即触发顺滑拖拽。
  * **边缘缩放层（最高优先级）**：当光标靠近边缘变为“伸缩双箭头”时，系统即作出操作承诺。此时按下左键**必须执行缩放**，中心按钮的 UI 显现绝不阻断此行为。

### 3. 多形态缩放控制标准 (Shape-Specific Scaling)
  控制点在新打开的背景为白色窗口下几乎看不清。
* **矩形形态（8 视口控制点）**
  * **四角定点（左上/左下/右上/右下）**：严格锁定当前画面的纵横比进行整体等比缩放，确保画面主体不变形。
  * **四边中点（上/下/左/右）**：执行横向或纵向的单向拉伸。该操作实为**改变视口裁剪区 (Viewport Cropping)**，需配合自适应填充（`object-fit: cover`），允许画面自由裁切为竖屏或宽屏，而非像素拉伸畸变。
* **圆形形态（4 对角控制点）**
  * **锚点重构**：彻底废除矩形边角，控制点精确吸附于圆弧的四个对角（45°、135°、225°、315°），控制点。
  * **强制等比**：拖拽任意对角控制点，必须保持圆心相对稳定，强制以 1:1 绝对等比例放大或缩小，杜绝任何椭圆形变。

### 4. 录制态工具条锁定规范 (Toolbar Locked State)
* 录制一旦启动，主工具条上所有控制按钮强制进入**禁用不可交互状态**（Disable/Pointer-events: none）。
* 禁用状态下，**必须保持当前的视觉标识**：正在录制的信息源高亮显示，未使用的信息源置灰显示。仅提供当前录制状态的可视化确认，绝不响应任何修改指令。

### 5. 级联生命周期强绑定 (Cascading Lifecycle Management)
* 确立“主工具条即主程序”的唯一控制论。
* 用户主动关闭主工具条窗口时，系统必须无条件触发级联销毁机制，同步关闭 PIP 窗口及其他所有附属 UI 进程，并即刻强制释放底层的麦克风与摄像头（MediaStreamTrack.stop()）等硬件独占锁。
